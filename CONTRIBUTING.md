# Как пополнять dev-base

Этот репо растёт по принципу: **столкнулся с задачей второй раз — пора писать сюда**.

## Куда что класть

- **`skills/`** — стандарты и инструкции уровня «как делать X в принципе».
  Пример: `github-repo-standard.md`, «как писать ТЗ», «как готовить релиз».
- **`prompts/`** — отлаженные текстовые промпты для LLM/AI-генераторов.
  Пример: `logo-til-kural.md`. Имя файла = `<сценарий>-<контекст>.md`.
- **`playbooks/`** — пошаговые сценарии «делай раз, делай два».
  Пример: «новый Solana-проект за 1 час», «деплой Plesk через FTP».
- **`modules/`** — переиспользуемые куски кода и конфигов.
  Пример: `next.config.standalone.ts`, `eslint.config.strict.mjs`, `gh-actions/ci-node.yml`.
- **`templates/`** — шаблоны документов.
  Пример: README, PR, Issue, CONTRIBUTING.

## Формат файла

Markdown. В шапке — однострочное описание, что это и когда применять. Если шаблон — добавить блок «как переиспользовать для другого проекта» в конце.

## Коммиты

По стандарту [`templates/conventional-commits.md`](templates/conventional-commits.md):
```
docs(skills): добавить стандарт оформления GitHub-репо
docs(prompts): добавить промпт для лого til-kural
chore(templates): обновить README.template — секция Why Stack
```

## Правило одного абзаца

Если описание скила/промпта в README ниже двух абзацев — этот файл не нужен. Либо допиши, либо удали.
