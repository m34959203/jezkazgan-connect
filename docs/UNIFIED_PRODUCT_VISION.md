## 1. Executive Summary

### Что меняется

| Параметр | Было (Jezkazgan Connect) | Стало (Afisha.kz) |
|----------|--------------------------|-------------------|
| География | 1 город (Жезказган) | Все города РК (17 областей + 3 города респ. значения) |
| Аудитория | ~60,000 пользователей | ~12,000,000 интернет-пользователей |
| Бизнесов | ~2,500 | ~500,000+ МСБ |
| Платформы | Web only | **Web + Android + iOS** |
| Название | Jezkazgan Connect | **Afisha.kz** |

### Ценностное предложение

> **"Единая афиша событий, скидок и бизнеса для всего Казахстана"**

**Для жителей:** Узнавай о событиях и скидках в своём городе
**Для бизнеса:** Продвигай свой бизнес миллионам казахстанцев

## 2. SWOT-анализ

### Сильные стороны (Strengths)

| # | Сильная сторона | Влияние |
|---|-----------------|---------|
| S1 | **Готовый UI/UX** — 85-90% фронтенда разработано | Быстрый запуск MVP |
| S2 | **Современный стек** — React, TypeScript, Tailwind | Масштабируемость, найм разработчиков |
| S3 | **Комплексное решение** — события + бизнес + скидки + сообщества | Конкурентное преимущество |
| S4 | **Freemium модель** — низкий порог входа для бизнеса | Быстрый рост базы |
| S5 | **Локальный фокус** — понимание рынка РК | Релевантный контент |

### Слабые стороны (Weaknesses)

| # | Слабая сторона | Риск | Решение |
|---|----------------|------|---------|
| W1 | **Нет бэкенда** — 0% готовности | Критичный | Разработка бэкенда — приоритет #1 |
| W2 | **Нет мобильных приложений** | Высокий | Параллельная разработка Android/iOS |
| W3 | **Нет тестирования** — 0% покрытия | Средний | Внедрить тесты с бэкендом |
| W4 | **Нет CI/CD** | Средний | Настроить с началом бэкенда |
| W5 | **Зависимость от контента** | Высокий | Партнёрства, UGC стимулирование |
| W6 | **Нет команды продаж** | Средний | Нанять после MVP |

### Возможности (Opportunities)

| # | Возможность | Потенциал |
|---|-------------|-----------|
| O1 | **Пустой рынок** — нет единой афиши для регионов РК | Высокий |
| O2 | **Рост МСБ** — гос. поддержка малого бизнеса | Высокий |
| O3 | **Digital-трансформация** — программа "Цифровой Казахстан" | Средний |
| O4 | **Молодая аудитория** — 50%+ населения до 35 лет | Высокий |
| O5 | **Партнёрство с акиматами** — интерес к цифровизации | Средний |
| O6 | **White-label** — продажа решения другим странам СНГ | Долгосрочный |

### Угрозы (Threats)

| # | Угроза | Вероятность | Митигация |
|---|--------|-------------|-----------|
| T1 | **Конкуренция от 2GIS/Yandex** | Средняя | Фокус на события и скидки (их слабое место) |
| T2 | **Копирование идеи** | Высокая | First-mover advantage, быстрый запуск |
| T3 | **Нехватка контента** | Высокая | Партнёрства с организаторами, импорт |
| T4 | **Сложность масштабирования** | Средняя | Модульная архитектура |
| T5 | **Экономический кризис** | Низкая | Freemium модель защищает |

## 4. Roadmap разработки

### 4.1 Обзор платформ

| Платформа | Приоритет | Технологии | Срок |
|-----------|-----------|------------|------|
| **Web** | P0 | React, TypeScript, Tailwind | Есть 85% |
| **Android** | P0 | React Native или Kotlin | Параллельно |
| **iOS** | P0 | React Native или Swift | Параллельно |
| **Backend** | P0 | Node.js/NestJS + PostgreSQL | Критично |

### 4.2 Детальный план разработки

#### PHASE 1: Backend + Web MVP (8-10 недель)

```
Неделя 1-2: Инфраструктура
├── PostgreSQL схема данных
├── NestJS/Express API структура
├── Docker + CI/CD (GitHub Actions)
├── Vercel (frontend) + Railway (backend)
└── Миграции, seed данные

Неделя 3-4: Core API
├── Auth: JWT, регистрация, вход, OAuth
├── Users CRUD + роли
├── Cities: мульти-город
├── Events CRUD + фильтры
└── Businesses CRUD + верификация

Неделя 5-6: Features API
├── Promotions (акции) CRUD
├── Communities + Groups
├── Collaborations
├── Search (Meilisearch)
└── Image upload (S3/Cloudflare R2)

Неделя 7-8: Интеграция + Платежи
├── Frontend-Backend интеграция
├── Kaspi Pay / PayBox
├── Подписки для бизнеса
├── Premium для пользователей
└── Уведомления (email, push)

Неделя 9-10: QA + Запуск
├── Тестирование (unit, e2e)
├── Нагрузочное тестирование
├── Security audit
├── Beta-тест (100 пользователей)
└── Production deploy
```

#### PHASE 2: Mobile Apps (10-12 недель, параллельно с Phase 1)

**Вариант A: React Native (рекомендуется)**
```
+ Единая кодовая база для Android и iOS
+ Быстрее разработка (1 команда)
+ Переиспользование логики с Web
- Немного хуже производительность
```

**Вариант B: Native (Kotlin + Swift)**
```
+ Лучшая производительность
+ Нативный UX
- Две команды разработки
- Дольше и дороже
```

**Рекомендация:** React Native для MVP, Native — в будущем при необходимости.

```
Неделя 1-3: Setup + Navigation
├── React Native + Expo
├── Навигация (React Navigation)
├── UI Kit (адаптация Web компонентов)
├── Интеграция с API
└── Авторизация

Неделя 4-6: Core Features
├── События: список, детали, фильтры
├── Бизнесы: каталог, профили
├── Акции: лента, детали
├── Геолокация + карта
└── Push-уведомления (Firebase)

Неделя 7-9: User Features
├── Профиль пользователя
├── Избранное
├── Сообщества
├── QR-коды (сканер + генератор)
└── Offline режим (базовый)

Неделя 10-12: Business Features + Release
├── Бизнес-панель в приложении
├── Создание акций/событий
├── Аналитика
├── App Store / Google Play публикация
└── Crash reporting (Sentry)
```

## 6. Риски и митигация

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Нехватка контента | Высокая | Высокое | Партнёрства с ивент-агентствами, импорт из соцсетей |
| Бизнесы не регистрируются | Высокая | Высокое | Бесплатный старт, личный онбординг топ-100 в каждом городе |
| Конкуренция от крупных игроков | Средняя | Высокое | Скорость, локальный фокус, фичи которых нет у конкурентов |
| Технические сбои | Средняя | Высокое | Мониторинг, резервирование, SLA |
| Недостаток финансирования | Средняя | Высокое | Lean-подход, ранняя монетизация, поиск инвестора |
