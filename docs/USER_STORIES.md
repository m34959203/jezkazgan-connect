# Часть 1: User Stories — Житель города (B2C)

## 1. Регистрация и аутентификация

### US-R01: Регистрация нового пользователя
**Как** житель Жезказгана,
**я хочу** зарегистрироваться на платформе,
**чтобы** получить доступ к персонализированным функциям (избранное, уведомления).

**Критерии приёмки:**
- [ ] Форма регистрации содержит поля: email, пароль, имя, телефон (опционально)
- [ ] Пароль должен быть минимум 8 символов
- [ ] При успешной регистрации пользователь автоматически авторизуется
- [ ] Отправляется email для подтверждения адреса
- [ ] Показывается сообщение об успешной регистрации

**Приоритет:** P0 (Critical)

---

### US-R02: Вход в аккаунт
**Как** зарегистрированный пользователь,
**я хочу** войти в свой аккаунт,
**чтобы** получить доступ к своим данным и избранному.

**Критерии приёмки:**
- [ ] Форма входа содержит поля: email, пароль
- [ ] Есть чекбокс "Запомнить меня"
- [ ] При неверных данных показывается понятная ошибка
- [ ] После успешного входа редирект на главную страницу
- [ ] Сессия сохраняется при закрытии браузера (если "Запомнить меня")

**Приоритет:** P0 (Critical)

---

### US-R03: Восстановление пароля
**Как** пользователь, забывший пароль,
**я хочу** восстановить доступ к аккаунту,
**чтобы** продолжить пользоваться платформой.

**Критерии приёмки:**
- [ ] Ссылка "Забыли пароль?" на странице входа
- [ ] Форма ввода email для восстановления
- [ ] Отправка письма со ссылкой на сброс пароля
- [ ] Ссылка действительна 24 часа
- [ ] Форма создания нового пароля

**Приоритет:** P1 (High)

---

### US-R04: Вход через социальные сети
**Как** пользователь,
**я хочу** войти через Google или Instagram,
**чтобы** не создавать отдельный пароль.

**Критерии приёмки:**
- [ ] Кнопки "Войти через Google" и "Войти через Instagram"
- [ ] При первом входе создаётся аккаунт автоматически
- [ ] Запрашивается только базовая информация (email, имя)
- [ ] Можно связать существующий аккаунт с соцсетью

**Приоритет:** P2 (Medium)

---

### US-R05: Выход из аккаунта
**Как** авторизованный пользователь,
**я хочу** выйти из аккаунта,
**чтобы** защитить свои данные на чужом устройстве.

**Критерии приёмки:**
- [ ] Кнопка "Выйти" в меню профиля
- [ ] Сессия завершается, токены удаляются
- [ ] Редирект на главную страницу
- [ ] Избранное и личные данные больше не доступны

**Приоритет:** P0 (Critical)

## 2. Просмотр и поиск событий

### US-E01: Просмотр списка событий
**Как** житель города,
**я хочу** видеть список предстоящих событий,
**чтобы** выбрать интересное мероприятие.

**Критерии приёмки:**
- [ ] Отображается сетка/список карточек событий
- [ ] Каждая карточка содержит: изображение, название, дату, место, цену
- [ ] События отсортированы по дате (ближайшие первыми)
- [ ] Отмечены "Featured" события
- [ ] Пагинация или бесконечный скролл при большом количестве

**Приоритет:** P0 (Critical)

---

### US-E02: Фильтрация событий по категории
**Как** пользователь,
**я хочу** фильтровать события по категориям,
**чтобы** быстро найти интересующий тип мероприятий.

**Критерии приёмки:**
- [ ] Доступны фильтры по 8 категориям: Концерты, Обучение, Семинары, Досуг, Спорт, Детские, Выставки, Другое
- [ ] Можно выбрать несколько категорий одновременно
- [ ] Список обновляется мгновенно при выборе
- [ ] Показывается количество событий в каждой категории
- [ ] Кнопка "Сбросить фильтры"

**Приоритет:** P0 (Critical)

---

### US-E03: Фильтрация событий по дате
**Как** пользователь,
**я хочу** фильтровать события по дате,
**чтобы** найти мероприятия на конкретный период.

**Критерии приёмки:**
- [ ] Быстрые фильтры: Сегодня, Эта неделя, Этот месяц, Все
- [ ] Возможность выбрать конкретную дату через календарь
- [ ] Возможность выбрать диапазон дат
- [ ] Фильтр применяется мгновенно
- [ ] Показывается выбранный период

**Приоритет:** P0 (Critical)

---

### US-E04: Фильтрация событий по цене
**Как** пользователь,
**я хочу** фильтровать события по стоимости,
**чтобы** найти бесплатные или доступные по цене мероприятия.

**Критерии приёмки:**
- [ ] Фильтры: Все, Бесплатные, Платные
- [ ] Для платных — слайдер диапазона цен
- [ ] Отображается минимальная и максимальная цена
- [ ] Бесплатные события отмечены специальным бейджем

**Приоритет:** P1 (High)

---

### US-E05: Поиск событий
**Как** пользователь,
**я хочу** искать события по ключевым словам,
**чтобы** быстро найти конкретное мероприятие.

**Критерии приёмки:**
- [ ] Поле поиска в шапке сайта
- [ ] Поиск по названию, описанию, организатору, месту, тегам
- [ ] Результаты появляются при вводе (автокомплит)
- [ ] Подсветка найденных слов в результатах
- [ ] История последних поисков

**Приоритет:** P1 (High)

---

### US-E06: Просмотр детальной информации о событии
**Как** пользователь,
**я хочу** видеть полную информацию о событии,
**чтобы** принять решение о посещении.

**Критерии приёмки:**
- [ ] Страница содержит: большое изображение, название, полное описание
- [ ] Дата, время, продолжительность (если указана)
- [ ] Место проведения с адресом и картой
- [ ] Информация о цене (диапазон, если есть)
- [ ] Информация об организаторе с логотипом
- [ ] Теги события
- [ ] Счётчики просмотров и сохранений
- [ ] Хлебные крошки для навигации

**Приоритет:** P0 (Critical)

---

### US-E07: Просмотр похожих событий
**Как** пользователь на странице события,
**я хочу** видеть похожие мероприятия,
**чтобы** найти альтернативы или дополнительные события.

**Критерии приёмки:**
- [ ] Блок "Похожие события" внизу страницы
- [ ] 3-4 события той же категории
- [ ] Исключено текущее событие
- [ ] Карточки кликабельны

**Приоритет:** P2 (Medium)

---

### US-E08: Просмотр событий на карте
**Как** пользователь,
**я хочу** видеть события на карте города,
**чтобы** найти мероприятия рядом со мной.

**Критерии приёмки:**
- [ ] Переключатель "Список / Карта"
- [ ] Маркеры событий на карте Жезказгана
- [ ] При клике на маркер — превью события
- [ ] Кластеризация при большом количестве событий
- [ ] Фильтры работают и в режиме карты

**Приоритет:** P2 (Medium)

## 3. Избранное и сохранения

### US-F01: Добавление события в избранное
**Как** авторизованный пользователь,
**я хочу** сохранить событие в избранное,
**чтобы** вернуться к нему позже.

**Критерии приёмки:**
- [ ] Кнопка-сердечко на карточке и странице события
- [ ] При клике событие добавляется/удаляется из избранного
- [ ] Визуальная индикация (заполненное сердце)
- [ ] Для неавторизованных — предложение войти
- [ ] Счётчик сохранений обновляется

**Приоритет:** P1 (High)

---

### US-F02: Просмотр списка избранного
**Как** авторизованный пользователь,
**я хочу** видеть все сохранённые события,
**чтобы** быстро к ним вернуться.

**Критерии приёмки:**
- [ ] Раздел "Избранное" в профиле пользователя
- [ ] Список сохранённых событий с карточками
- [ ] Возможность удалить из избранного
- [ ] Сортировка по дате добавления или дате события
- [ ] Индикация прошедших событий

**Приоритет:** P1 (High)

---

### US-F03: Сохранение бизнеса в избранное
**Как** пользователь,
**я хочу** сохранять понравившиеся бизнесы,
**чтобы** быстро находить их в будущем.

**Критерии приёмки:**
- [ ] Кнопка сохранения на карточке бизнеса
- [ ] Отдельный раздел "Избранные места" в профиле
- [ ] Уведомления о новых акциях избранных бизнесов

**Приоритет:** P2 (Medium)

### US-BF01: Просмотр текущего тарифа
**Как** владелец бизнеса,
**я хочу** видеть информацию о своём тарифе,
**чтобы** знать доступные функции.

**Критерии приёмки:**
- [ ] Текущий тариф в панели управления
- [ ] Дата следующего платежа
- [ ] Использование лимитов (если есть)
- [ ] Кнопка "Улучшить тариф"

**Приоритет:** P1 (High)

---

### US-BF02: Оплата подписки
**Как** владелец бизнеса,
**я хочу** оплатить подписку онлайн,
**чтобы** получить доступ к premium-функциям.

**Критерии приёмки:**
- [ ] Выбор тарифа: Бизнес (5000₸/мес) или VIP (15000₸/мес)
- [ ] Выбор периода: месяц, 3 месяца, год (со скидкой)
- [ ] Оплата картой или Kaspi
- [ ] Мгновенная активация после оплаты
- [ ] Email с чеком

**Приоритет:** P1 (High)

---

### US-BF03: Отмена подписки
**Как** владелец бизнеса,
**я хочу** отменить подписку,
**чтобы** прекратить автоплатежи.

**Критерии приёмки:**
- [ ] Кнопка "Отменить подписку" в настройках
- [ ] Подтверждение с указанием, до какой даты действует
- [ ] Подписка остаётся активной до конца оплаченного периода
- [ ] Переход на бесплатный тариф после окончания

**Приоритет:** P1 (High)

---

### US-BF04: Просмотр истории платежей
**Как** владелец бизнеса,
**я хочу** видеть историю платежей,
**чтобы** контролировать расходы.

**Критерии приёмки:**
- [ ] Список всех платежей с датами и суммами
- [ ] Статус каждого платежа
- [ ] Скачивание чеков/актов
- [ ] Фильтр по периоду

**Приоритет:** P2 (Medium)

# Часть 3: User Stories — Модератор/Админ

### US-A01: Модерация контента
**Как** модератор,
**я хочу** проверять новый контент,
**чтобы** обеспечить качество платформы.

**Критерии приёмки:**
- [ ] Очередь на модерацию: события, акции, отзывы
- [ ] Кнопки: Одобрить, Отклонить, Запросить изменения
- [ ] Комментарий для автора при отклонении
- [ ] Автоматическое уведомление автора

**Приоритет:** P0 (Critical)

---

### US-A02: Управление пользователями
**Как** админ,
**я хочу** управлять аккаунтами пользователей,
**чтобы** поддерживать порядок.

**Критерии приёмки:**
- [ ] Поиск пользователей по имени/email
- [ ] Просмотр профиля и активности
- [ ] Блокировка/разблокировка аккаунта
- [ ] Изменение роли пользователя
- [ ] Удаление аккаунта

**Приоритет:** P1 (High)

---

### US-A03: Верификация бизнесов
**Как** модератор,
**я хочу** проверять заявки на верификацию,
**чтобы** подтверждать легитимность бизнесов.

**Критерии приёмки:**
- [ ] Очередь заявок на верификацию
- [ ] Просмотр загруженных документов
- [ ] Одобрение или отклонение с комментарием
- [ ] Присвоение статуса "Верифицирован"

**Приоритет:** P1 (High)

---

### US-A04: Аналитика платформы
**Как** админ,
**я хочу** видеть общую аналитику платформы,
**чтобы** принимать решения о развитии.

**Критерии приёмки:**
- [ ] Количество пользователей (всего, новых, активных)
- [ ] Количество бизнесов (всего, верифицированных)
- [ ] Количество событий и акций
- [ ] Графики роста
- [ ] Финансовые метрики (MRR, доход)

**Приоритет:** P1 (High)

## Формат User Story

Все user stories написаны в формате:

```
**Как** [роль пользователя],
**я хочу** [действие/функция],
**чтобы** [выгода/цель].

**Критерии приёмки:**
- [ ] Критерий 1
- [ ] Критерий 2
...

**Приоритет:** P0/P1/P2/P3
```
